handle\+Key\+Down function\+: a function to trigger when an on\+Key\+Down event fires. Meant to be provided to Workspace User\+Inputs. 

When on\+Key\+Down fires, the handle\+Key\+Down function ensures that the enter key cannot insert a carriage return in the text area like normal, that the delete key does not delete a character, that the tab key does not refocus to the next focusable area, and that shift+tab does not focus to the previous focusable area. Instead, the enter key inserts a new line into the Workspace, the delete key deletes a line from the Workspace, the tab key prepends a \textquotesingle{}\textbackslash{}t\textquotesingle{} into the focused textarea, and shift+tab removes a prepended \textquotesingle{}\textbackslash{}t\textquotesingle{} from the focused textarea if such a \textquotesingle{}\textbackslash{}t\textquotesingle{} exists.


\begin{DoxyParams}{Parameters}
{\em event} & The event that was told to trigger this function. \\
\hline
{\em list} & Any array, but this array should be the block\+List. \\
\hline
{\em index} & The current Workspace line in focus. \\
\hline
{\em update\+Block\+List} & The update function to update the block\+List once a change is made to the block\+List.\\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
None. Any changes made in the function will be handled by update\+Block\+List.
